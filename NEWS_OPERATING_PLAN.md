# 臻宝每日简讯更新策略

## 目标

把网站从静态新闻页升级为可持续滚动更新的个人信息仪表盘。核心原则是：新闻新、来源稳、重复少、栏目清楚、用户不用等。

## 栏目命名

- 每日热搜 -> 新闻
- 股市行情 -> 股市
- 娱乐热点 -> 娱乐
- 河南足球 -> 足球

## 新闻栏目

### 子栏目

- 国内
- 国际
- AI 科技

### 更新频率

- GitHub Actions 每 3 小时运行一次。
- 每个子栏目保持 100 条有效新闻。
- 只保留 48 小时内的新闻。超过 48 小时的新闻进入淘汰池，不再进入首屏和更多列表。

### 排序规则

前端展示继续使用抗疲劳排序：

```text
最终分数 = 热度 + 新鲜度 + 来源权威 + 稳定随机 - 曝光惩罚 - 相似新闻惩罚
```

抓取端排序以发布时间和收录时间为主：

```text
publishedAt 优先，缺失时使用 collectedAt；同一事件多来源保留权威来源或摘要更完整的一条。
```

### 国际和 AI 科技来源

优先使用非中国本地媒体和机构来源，抓取后统一翻译标题与摘要为中文，并保留原始来源名。建议来源池：

- 国际：Reuters、Associated Press、BBC、The Guardian、Al Jazeera、NPR、DW、France 24
- AI 科技：MIT Technology Review、The Verge、TechCrunch、Ars Technica、Wired、VentureBeat AI、Google Blog、OpenAI Blog、Anthropic News、Microsoft AI Blog、NVIDIA Blog

### 审核标准

- 必须有标题、来源、链接、发布时间或收录时间。
- 标题不能是广告、软文、彩票、低质自媒体标题。
- 国际和 AI 科技优先原始报道、通讯社、官方博客和高信誉科技媒体。
- 同一事件最多保留 1-2 条，避免列表被同一主题刷屏。

## 股市栏目

### 指数数据

- 页面端 A 股/美股交易时段：每 1 分钟刷新一次。
- 页面端非交易时段：每 5 分钟检查一次。
- 跑马灯行情：每 1 分钟刷新一次。
- 每次成功刷新都更新 `stockDataUpdateTime`。
- 如果 API 校验失败，不覆盖旧数据。

### 股市新闻

- 保持 40-60 条高质量财经新闻。
- 更新频率：每 3 小时，和新闻栏目同一条 pipeline 即可。
- 优先非中国本地财经来源：Reuters Business、CNBC、MarketWatch、Bloomberg（如可访问）、Yahoo Finance、Financial Times（标题/摘要可用时）、WSJ（标题/摘要可用时）。
- 标题和摘要翻译成中文，保留来源。

### AI 观察池

- 单股观察：每 15 分钟复核一次，基于盘面涨跌、新闻热度、行业景气、政策催化、风险项打分。
- 板块预测：每 30 分钟复核一次，输出强势/关注/稳健标签。
- 预测由规则评分引擎生成，不给直接买卖指令，页面明确提示“不构成投资建议”。

## 娱乐栏目

### 榜单排行

- 电影、电视剧、综艺榜单每 2 天更新一次。
- 电影优先猫眼、淘票票、豆瓣、IMDb/Letterboxd 公开榜单。
- 电视剧/综艺优先猫眼、骨朵、灯塔、豆瓣、平台热度榜。

### 娱乐新闻

- 来源以用户提供的源 HTML 为主。
- 过滤低质八卦、重复生图、无信息量标题。
- 保持 100 条以内，两天内新闻优先。

## 足球栏目

### 信息结构

- 中超积分榜
- 河南队近期赛程
- 河南队历史战绩
- 河南队动态
- 中超动态

### 更新策略

- 积分榜：比赛日后更新；常规每天至少检查一次。
- 赛程：赛季表相对稳定，每周检查一次；若足协调整赛程则立即更新。
- 历史战绩：河南队比赛结束后更新。
- 河南队动态：每天 2-3 次，比赛日前后加密。
- 中超动态：每天 2-3 次，重点覆盖积分争冠、伤停、转会、裁判、足协公告。

### 推荐来源

- 河南足球俱乐部官方渠道
- 中超联赛官方/中国足协公告
- 懂球帝、直播吧、搜狐体育、新浪体育、腾讯体育
- 正观新闻、大河报等河南本地媒体

## 专属栏目

### 当前问题

当前实现把用户兴趣词写入 GitHub 仓库，再触发 GitHub Actions 生成 `exclusive_news_arrays.js`。这个方式能跨设备，但有明显缺点：

- 前端暴露 GitHub token 风险高。
- 用户添加兴趣词后需要等 workflow 完成。
- 多用户冲突、失败重试、同步状态都不够自然。
- 静态网站不适合直接承载写操作。

### 简化方案

第一阶段先取消“必须同步 GitHub 才可用”的体验：

- 兴趣词先保存在本地 `localStorage`。
- 页面立即从已收录的全量新闻池中做本地匹配。
- 匹配规则：标题/摘要/来源命中关键词，按新鲜度和热度排序。
- 如果匹配少，再提示用户“已保存，下一次全站更新后会补充更多”。

第二阶段再增加轻量服务端：

- 使用 GitHub Actions 只做公共新闻池更新，不为每个用户生成静态专属数组。
- 用户专属推荐在浏览器本地完成，避免写仓库。
- 如需跨设备同步，建议以后接 Supabase/Cloudflare KV/Workers，而不是前端写 GitHub。

## 设置页

设置页应保留必要能力，减少解释性文字：

- 账号/昵称
- 主题外观
- 推送开关
- 关怀模式
- 数据更新时间
- 清除本地阅读记录/兴趣词

6. 股市数据刷新时间写入页面，并按交易时段区分频率。

## UI 优化方向

- 当前先保留卡片式信息流，避免大改导致阅读路径变乱。
- 下一阶段可把首页改成“顶部快讯 + 三栏摘要 + 底部导航”的仪表盘：新闻首屏更短，股市/足球/专属入口更清晰。
- 足球页继续走信息面板化：积分榜和河南队关注置顶，赛程/战绩并列，动态新闻分河南队和中超全局。
- 设置页减少说明文字，增加“清除阅读记录 / 清除兴趣词 / 数据更新时间”三个实用操作。
