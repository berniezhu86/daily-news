# 中超其他球队新闻自动更新

## 功能说明

自动搜索最新的中超其他球队新闻（上海海港、上海申花、北京国安、成都蓉城、山东泰山等），并更新到网页的"中超其他球队动态"模块。

## 自动化任务配置

- **任务名称**：中超其他球队新闻自动更新
- **任务ID**：automation-1782434415524
- **执行频率**：每天3次（6:00、12:00、18:00）
- **状态**：已启用（ACTIVE）

## 执行流程

1. **搜索最新新闻**
   - 使用 WebSearch 搜索关键词："中超 第16轮 最新新闻"、"上海申花 最新"、"上海海港 最新"等
   - 只保留发布时间在最近3天内的新闻

2. **提取新闻信息**
   - 标题（title）
   - 来源（source）
   - 摘要（summary）
   - 发布时间（time）
   - 热度（heat，1-10）

3. **更新网页**
   - 读取 `index.html`
   - 找到 `mockCslOtherTeams` 数组
   - 用最新新闻替换

4. **提交到 GitHub**
   - `git add index.html`
   - `git commit -m "auto: 更新中超其他球队新闻"`
   - `git push origin main`

## 新闻数据格式

```javascript
const mockCslOtherTeams = [
  {
    team: "上海申花",          // 球队名称
    title: "新闻标题",          // 新闻标题
    source: "搜狐体育",         // 新闻来源
    summary: "新闻摘要",        // 新闻摘要
    time: "2026-06-26 06:32",  // 发布时间（必须）
    heat: 8                     // 热度（1-10，越高越重要）
  },
  // ... 更多新闻
];
```

## 手动触发

如果需要立即更新新闻，可以：

1. 打开 WorkBuddy 自动化管理页面
2. 找到"中超其他球队新闻自动更新"任务
3. 点击"立即执行"按钮

## 查看执行历史

自动化任务的执行历史存储在 SQLite 数据库中：
- 数据库路径：`~/.workbuddy/workbuddy.db`
- 表名：`automation_runs`

## 修改执行频率

如果需要修改执行频率（例如改为每天1次），可以：

1. 打开 WorkBuddy 自动化管理页面
2. 找到"中超其他球队新闻自动更新"任务
3. 点击"编辑"按钮
4. 修改 `rrule` 字段

示例：
- 每天1次（早上8点）：`RRULE:FREQ=DAILY;BYHOUR=8`
- 每周1次（周一早上8点）：`RRULE:FREQ=WEEKLY;BYDAY=MO;BYHOUR=8`

## 暂停/启用任务

- **暂停**：将 `status` 改为 `PAUSED`
- **启用**：将 `status` 改为 `ACTIVE`

## 注意事项

1. **发布时间验证**：确保每条新闻的 `time` 字段准确，避免使用旧闻
2. **新闻数量**：保持 10-15 条新闻，避免过多或过少
3. **热度设置**：重要新闻设置 `heat >= 7`，会在列表中置顶显示
4. **GitHub 提交**：每次更新会自动提交到 GitHub，EdgeOne Pages 会自动部署

## 故障排查

如果自动化任务执行失败，可以：

1. 查看执行历史，检查错误信息
2. 手动执行一次，观察是否有错误
3. 检查 WebSearch 是否正常访问
4. 检查 GitHub 仓库权限
